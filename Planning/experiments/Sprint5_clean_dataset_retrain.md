**Date**: 2026-05-11
**Status**: complete (DL portion); classical portion done in parallel by Person A
Related: [[Validation_and_Verification]] (§3.5 — Diagnostic 5), [[experiments/Sprint3_classical_on_full]], [[Tutor_Meeting_Brief]], [[Results_Summary]]

> **Headline**: removing the benchmark's previously-undocumented duplicate images costs **17–21 percentage points** of macro-F1 on the DL backbones and **8 pp** on classical. **Classical SVM now statistically dominates both DL backbones on every class and every pairwise comparison (McNemar's p ≤ 1.9 × 10⁻¹⁵).** The Sprint 3 invalidation chain is now superseded by a single empirically-demonstrated leakage gap.

## Trigger

On **2026-05-07**, Sandhya posted on Canvas confirming the Islam et al. 2022 dataset contains duplicate images that "resulted in some images appearing in both the training and testing datasets, which can lead to an artificially high testing accuracy and a less reliable evaluation of the system." She released a deduplicated dataset for students to re-run on. **This independently confirms [[Validation_and_Verification]] §3.5 (Diagnostic 5, 2026-04-30)** which surfaced 96 / 1,867 (5.1 %) bit-identical train↔test pairs via MD5 hashing 2 days earlier.

## Method

Same training protocols as Sprint 2 and Sprint 4 — only the training and test data change. No hyperparameter retuning, no protocol changes. Strict matched-protocol comparison so any metric movement is purely the leakage effect.

| Parameter | Value | Same as Sprint 2 / Sprint 4? |
|---|---|---|
| Dataset | `Updated_Dataset/CT-KIDNEY-DATASET-Normal-Cyst-Tumor-Stone_unique/` (11,929 images, 0 MD5 collisions per re-run of [[Validation_and_Verification]] Diag 5) | × — new (was 12,446 images with 260 within-train + 96/115/22 cross-split MD5 collisions) |
| Split | `split_full.csv` (regenerated by Person A with `group_size=50` filename-bucket pseudo-patient stratification) — 8,146 train / 1,895 val / 1,888 test | × — new (was 8,712 / 1,867 / 1,867 random stratified) |
| EfficientNet-B0 protocol | Two-stage, 224×224, batch 32, AdamW wd=1e-4, stage2_unfreeze_blocks=2 | ✓ |
| ConvNeXt V2 protocol | Two-stage, 384×384, batch 32, AdamW wd=0.05, stochastic depth 0.3, stage2_unfreeze_blocks=1 | ✓ |
| Compute | Colab Pro+ A100 | ✓ |

The training data dropped from 8,712 to 8,146 (−566) and the test set went from 1,867 to 1,888 (slight reshuffling under the new group-aware split). The classical predictions are from Person A's clean SVM run on the same `split_full.csv` test partition, so all three pipelines are evaluated on the *same n=1,888 test images* — paired McNemar's is valid.

## Headline results — leak-inclusive vs leak-free

`Results/sprint5_clean_vs_leaky.json`:

| Pipeline | Leaky macro-F1 | Clean macro-F1 | **Δ gap** |
|---|---|---|---|
| EfficientNet-B0 (full) | 0.9819 (n=1,867) | **0.7679** (n=1,888) | **−0.2139 (−21.4 pp)** |
| ConvNeXt V2 Base (full) | 0.9953 (n=1,867) | **0.8219** (n=1,888) | **−0.1734 (−17.3 pp)** |
| Classical SVM (full) — partner | 0.9897 *(XGB pre-Sprint-5)* | **0.9091** (n=1,888) | −0.0806 (−8.1 pp) |

Leaky and clean test sets are **not** the same images (n=1,867 vs n=1,888), so the gap is directional rather than paired. But the magnitude is unambiguous: deep CNNs lost **2–2.5×** the accuracy that classical did. **The deep models had the spare capacity to memorise the duplicates; classical's handcrafted-feature pipeline did not.** Yagis et al. 2021 predicted 29–55 % inflation in 2D-CNN MRI studies due to slice-level leakage; we observe 17–21 % — same order of magnitude, lower bound because our patient grouping is approximate.

## Per-class F1 on the CLEAN test set (n=1,888)

| Class | Classical SVM | EfficientNet-B0 | ConvNeXt V2 Base |
|---|---|---|---|
| Cyst | **0.961** | 0.852 | 0.894 |
| Normal | **0.939** | 0.859 | 0.902 |
| Stone | **0.799** | 0.610 | 0.698 |
| Tumor | **0.937** | 0.751 | 0.794 |

**Classical wins every class on the clean test set.** The previous "Tumor is solved at 100 %" finding from leaky XGBoost is now revealed as a memorisation artefact — the genuine Tumor F1 on novel images is 0.75 (EffNet) / 0.79 (ConvNeXt) / 0.94 (classical). Stone remains the universally weakest class but the gap to other classes has narrowed for classical (0.799 vs ~0.94) and widened for DL (0.61 / 0.70 vs ~0.85–0.90).

## Paired McNemar's on the CLEAN n=1,888 test set

`Results/sprint5_clean_vs_leaky.json` `pairwise_mcnemar_on_clean_test`:

| Comparison | Both correct | Only A wrong | Only B wrong | Both wrong | Discordant | p-value |
|---|---|---|---|---|---|---|
| **Classical vs EfficientNet-B0** | 1,426 | 87 | **326** | 49 | 413 | **1.1 × 10⁻³¹** (Classical dominates) |
| **Classical vs ConvNeXt V2** | 1,529 | 83 | **223** | 53 | 306 | **1.9 × 10⁻¹⁵** (Classical dominates) |
| EfficientNet-B0 vs ConvNeXt V2 | 1,374 | 238 | 139 | 137 | 377 | **4.5 × 10⁻⁷** (ConvNeXt > EffNet — same direction as leaky) |

**All three pairs are highly significant on clean data.** The previous "all three are tied" finding from leaky paired McNemar's (p > 0.05 for classical-vs-DL on leaky) was a leakage artefact. The true ranking on clean data is:

> **Classical SVM > ConvNeXt V2 Base > EfficientNet-B0** — all gaps statistically significant.

## Failure-mode persistence (Sprint 3 paradigm-stable claim, re-tested on clean)

The Sprint 3 finding that **only DL pipelines make `Cyst → Stone` errors** holds on clean data — and the asymmetry is *larger*:

| Pipeline | `Cyst → Stone` errors on leaky test | `Cyst → Stone` errors on clean test |
|---|---|---|
| Classical | 0 (XGB at leaky) | **13** (SVM at clean) |
| EfficientNet-B0 | 9 | **70** |
| ConvNeXt V2 | 3 | **35** |

Both classical and DL became worse at distinguishing Cyst from Stone on the clean test set (no more memorisation crutch), but DL got disproportionately worse. The relative asymmetry (DL ≫ classical on Cyst→Stone errors) survives on clean data.

## What this means for the paper — final narrative

The four-step invalidation chain from Sprints 3 + addenda is now **subordinate** to a single empirically-grounded headline finding. The paper's central narrative becomes:

> *"We trained classical handcrafted-feature ML (108→138-dim texture features + SVM) and two transfer-learned CNN backbones (EfficientNet-B0 and ConvNeXt V2 Base) on the Islam et al. 2022 kidney CT benchmark and observed near-saturation accuracy (97–99 % macro-F1) consistent with prior published work. We then surfaced via MD5 hashing (independently confirmed by the dataset maintainer 2 days later) that the benchmark contains 5.1 % bit-identical train↔test image duplication. Re-training all three pipelines on the maintainer-released deduplicated dataset reveals that **deep CNNs lost 17–21 percentage points of macro-F1 while classical handcrafted features lost only 8**. On the clean test set, classical SVM statistically dominates both CNN backbones (paired McNemar's p ≤ 1.9 × 10⁻¹⁵ for both pairs) and wins every per-class F1 comparison. We argue that on saturated medical-imaging benchmarks, the headline figures of every reported result in the literature should be treated as upper bounds until train-test leakage is explicitly diagnosed via cryptographic hashing — and that classical handcrafted-feature pipelines are substantially more robust to slice-level duplication artefacts than transfer-learned CNNs because they lack the capacity to memorise individual images."*

That paragraph is the paper's abstract.
